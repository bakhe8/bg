# مشروع bg

هذا المستودع أصبح يحتوي على ثلاث مناطق واضحة بعد تنظيمه:

| المسار | الوظيفة |
|--------|---------|
| `exceltojson/` | محرك تحويل ملفات Excel إلى JSON، مع واجهة سطر أوامر، اختبارات، وواجهة ويب (`python -m exceltojson.web_app`) التي تقدم صفحة الطباعة وتوفّر API `/api/convert`. |
| `jsontoweb/` | تطبيق الطباعة الرسمي (A4) وكل الأصول المرتبطة به (`assets/` تضم الخلفيات، الخطوط، الضمانات الافتراضية، قوالب البنوك). |
| `archives/` | نسخ JSON المؤرشفة تلقائياً لكل ملف Excel يتم تحويله عبر واجهة الويب. |
| `legacy/` | النماذج والمحاولات القديمة (`gpt/`, `js/`, `JsontoHTML/`, `test2/`). موجودة للرجوع إليها ولا تؤثر على العمل الحالي. |

## كيفية التشغيل

1. **تثبيت الاعتمادات** (مرة واحدة):
   ```bash
   pip install -r requirements.txt  # أو pip install flask pandas openpyxl
   ```
2. **تشغيل الواجهة الموحّدة**:
   ```bash
   python -m exceltojson.web_app
   ```
   ومن ثم افتح `http://127.0.0.1:5000` لاستخدام صفحة الطباعة (من مجلد `jsontoweb/`)، تحميل ملفات Excel، أو استخراج كل السجلات PDFs.

3. **استخدام أداة CLI التقليدية** (اختياري):
   ```bash
   python exceltojson/excel_to_json_converter.py data.xlsx --sheet all -o output.json
   ```

## ملاحظات تنظيمية

- يمكن تنظيف مجلد `archives/` دورياً أو نقله إلى تخزين خارجي عند تضخم الحجم.
- أي تطوير تجريبي جديد يوضع تحت `legacy/` أو في فرع git منفصل، مع دمج النتائج في `jsontoweb/` أو `exceltojson/` عند اعتمادها.
- التزم بتحميل الأصول المشتركة (الخطوط، القوالب) داخل `jsontoweb/assets/` حتى تبقى الواجهة مستقرة.
