# مشروع BGLApp

التنظيم الحالي يقسم المستودع إلى ثلاث طبقات رئيسية لتسهيل التطوير والتدرّج بين النسخ القديمة والجديدة:

```
BGLApp/
├ legacy/                ← مرجع ثابت لكل ما كان يعمل مسبقاً
│   ├ bgold/bg.zip
│   ├ exceltojson/
│   ├ JsontoHTML/
│   ├ test/
│   └ anythingelse/
│
├ transition/            ← مساحة العمل المرحلية
│   ├ backend/           ← محول Excel → JSON (ExcelToJsonConverter)
│   ├ web/               ← تطبيق Flask (app.py) الذي يربط الواجهة مع المحول ويعرض API
│   ├ integration_tests/ ← اختبارات الدمج
│   └ docs/              ← مستندات التصميم والمواصفات
│
└ main/                  ← الصورة المستهدفة للمشروع
    ├ bg_ui/             ← واجهة الخطابات النهائية (نسخة jsontoweb)
    ├ api/               ← الربط المستقبلي بين الواجهة والخدمات
    └ data/archives/     ← مخرجات JSON/PDF المؤرشفة
```

## طريقة التشغيل الحالية

1. **تثبيت الاعتمادات** (مرة واحدة):
   ```bash
   pip install -r requirements.txt
   ```

2. **تشغيل خادم Flask (مسار transition)**:
   ```bash
   python -m transition.web.app
   ```
   بعد التشغيل افتح `http://127.0.0.1:5000` لاستخدام واجهة `main/bg_ui/`، رفع ملفات Excel، أو استخراج جميع السجلات بصيغة PDF. يتم حفظ أي JSON ناتج داخل `main/data/archives/` تلقائياً.

3. **تشغيل أداة CLI القديمة** (من نسخة legacy عند الحاجة):
   ```bash
   python legacy/exceltojson/excel_to_json_converter.py data.xlsx --sheet all -o output.json
   ```
